Chapter 01

The Architecture

Most "second brain" setups die the moment you close your laptop. Your notes are local. Your automations require the machine to be awake. Your AI agents can't reach your knowledge base from anywhere else.

This guide solves that. By the end, you'll have:

  • An Obsidian vault accessible from any device over a private mesh network
  • A Telegram bot that can read, write, and search your vault from your phone
  • AI chat (Gemini or Claude) that answers questions using your vault as context
  • Always-on infrastructure that survives reboots and laptop sleep
Phone (anywhere) → Telegram message → Bot on Proxmox (always-on home server) → Tailscale mesh VPN (encrypted) → Mac (Obsidian vault + services) Components: ┌─────────────────────────────────────────────┐ │ MAC (Tailscale: 100.x.x.x) │ │ ├── Obsidian + REST API (:27124) │ │ ├── HTTP Proxy (:28124) ← bridges HTTPS │ │ └── FastAPI server (:8000) ← optional │ ├─────────────────────────────────────────────┤ │ PVE CONTAINER (Tailscale: 100.y.y.y) │ │ ├── picnic-bot (systemd) │ │ │ ├── Telegram handler │ │ │ ├── AI chat (Gemini/Claude) │ │ │ └── Vault search (pgvector) │ │ └── Always-on, auto-restarts │ └─────────────────────────────────────────────┘
Why not just run the bot on my Mac?

Because your Mac sleeps. The whole point is always-on access. A $200 mini PC running Proxmox uses ~15W and runs 24/7. Your bot never misses a message.

Chapter 02

Tailscale Mesh — Zero-Config VPN

Tailscale creates an encrypted mesh network between your devices. No port forwarding, no firewall rules, no dynamic DNS. Each device gets a stable IP that works from anywhere.

Install on Mac

Download from tailscale.com or install via Homebrew:

brew install tailscale
tailscale up

Sign in with your Google/GitHub account. Your Mac gets an IP like 100.121.35.109.

Install on Proxmox (LXC Container)

Inside your container:

# Install Tailscale
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up

# Verify mesh
tailscale status
# Should show both your Mac and the container
LXC Container Requirements

Your Proxmox container needs nesting=1 and keyctl=1 in its features for Tailscale to work. Set this in the container's options panel or in /etc/pve/lxc/<id>.conf.

Verify Connectivity

# From your PVE container, ping your Mac
tailscale ping 100.121.35.109
# pong from babys-macbook-air via 192.168.0.x in 8ms

Chapter 04

The Obsidian REST API Bridge

The Obsidian Local REST API plugin exposes your vault over HTTPS on localhost:27124. The problem: it uses a self-signed certificate and only listens on localhost.

Solution: a tiny Python HTTP proxy that bridges the gap.

obsidian-proxy.py

import http.server, http.client, ssl

class ObsidianProxyHandler(http.server.BaseHTTPRequestHandler):
    def _proxy(self):
        ctx = ssl.create_default_context()
        ctx.check_hostname = False
        ctx.verify_mode = ssl.CERT_NONE
        conn = http.client.HTTPSConnection(
            "127.0.0.1", 27124, context=ctx
        )
        length = int(self.headers.get("Content-Length", 0))
        body = self.rfile.read(length) if length else None
        headers = {
            k: v for k, v in self.headers.items()
            if k.lower() != "host"
        }
        conn.request(self.command, self.path,
                     body=body, headers=headers)
        resp = conn.getresponse()
        self.send_response(resp.status)
        for k, v in resp.getheaders():
            if k.lower() not in ("transfer-encoding",):
                self.send_header(k, v)
        self.end_headers()
        self.wfile.write(resp.read())
        conn.close()

    do_GET = do_PUT = do_POST = do_PATCH = do_DELETE = _proxy
    def log_message(self, *args): pass

httpd = http.server.HTTPServer(
    ("0.0.0.0", 28124), ObsidianProxyHandler
)
httpd.serve_forever()

Make it persistent (macOS launchd)

Save this as ~/Library/LaunchAgents/com.obsidian.proxy.plist:

<?xml version="1.0" encoding="UTF-8"?>
<plist version="1.0">
<dict>
    <key>Label</key>

    <key>ProgramArguments</key>

    </array>

clients/obsidian.py — Vault CRUD

import httpx
from datetime import datetime

class ObsidianClient:
    def __init__(self, host, port, api_key):
        self.base = f"http://{host}:{port}"
        self.headers = {
            "Authorization": f"Bearer {api_key}"
        }

    async def create_inbox_note(self, text):
        now = datetime.now()
        path = f"inbox/{now:%Y-%m-%d_%H%M%S}.md"
        content = f"""---
date: {now:%Y-%m-%d}
type: inbox
tags: [quick-capture, telegram]
---

{text}"""
        async with httpx.AsyncClient(timeout=15) as c:
            await c.put(
                f"{self.base}/vault/{path}",
                content=content,
                headers={**self.headers,
                         "Content-Type": "text/markdown"},
            )
        return path

handlers/chat.py — AI with vault context

from google import genai

async def handle_chat(message, vault_search, config):
    # 1. Search vault for relevant context
    results = await vault_search(message, limit=3)
    context = "\n".join(
        f"[[{r['file_path']}]]: {r['content'][:500]}"
        for r in results
    )

    # 2. Call AI with vault context
    client = genai.Client(api_key=config.google_api_key)
    response = client.models.generate_content(
        model="gemini-2.0-flash",
        contents=f"Context:\n{context}\n\nQuestion: {message}",
    )
    return response.text

Chapter 06

AI Chat with Vault Context

The magic: when you send the bot a free-text message, it searches your vault for relevant context, then sends that context + your question to an AI model. The AI answers using your own notes as knowledge.

For semantic search, we use Supabase pgvector with Google Gemini embeddings (768-dim). But you can swap in any embedding provider.

How the search works

  1. Your vault is chunked and embedded (one-time indexing)
  2. Each chunk is stored in Supabase with its vector
  3. When you search, your query is embedded and compared via cosine similarity
  4. Top 3-5 matching chunks are returned as context
Cost

Gemini embeddings are essentially free at small scale. Gemini Flash for chat costs ~$0.01 per response. You could run this for months on the free tier.

Chapter 07

Systemd — Making It Bulletproof

The bot needs to survive container reboots, crashes, and network blips.

# /etc/systemd/system/picnic-bot.service
[Unit]
Description=AI Second Brain Bot
After=network-online.target tailscaled.service
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/opt/picnic-bot
ExecStart=/opt/picnic-bot/.venv/bin/python3 -m picnic_bot.main
Restart=always
RestartSec=10
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
# Enable and start
systemctl daemon-reload
systemctl enable picnic-bot
systemctl start picnic-bot

# Check status
systemctl status picnic-bot
# ● picnic-bot.service - AI Second Brain Bot
#   Active: active (running)
Monitoring

View logs with journalctl -u picnic-bot -f. The bot logs every Telegram command and API call. If it crashes, systemd restarts it within 10 seconds.
